## 2. 分类器输出

```
personal_pity     : 52   ← 个人保底实测（金矿）
mechanic_qna      : 43   ← "80只是怎么算的"机制问答（噪音）
other             : 50   ← 杂项
ball_count        : 11   ← "抓了X只" 但语义是球数/总精灵数
genshin_noise     : 4    ← 原神 120 抽混入
```

**分类 heuristic**:
- `genshin_noise`: 含 `原神 / up角色 / 卡池 / 限定六星`
- `mechanic_qna`: 含 `保底.{是,为,分,怎么,算}` 模板 OR 含 `?` + `保底`
- `personal_pity`: 含 `我/俺/咱` + 动词 (`刷/抓/肝/出/才出`) + 上下文有 `污染/噩梦`
- `ball_count`: personal + `球` 上下文但缺 `污染` 上下文

## 3. Personal pity 分布

```
n stats: count=52, mean=308.6, median=50, max=2388

[   1-  9 ]: 15   ← "我第1只就是异色" — 部分仍是 showoff 噪音
[  10- 29 ]: 7
[  30- 59 ]: 8
[  60- 90 ]: 10   ← 保底带, 真实抽样
[  91-150 ]: 1    ← 小空缺
[ 151-500 ]: 1
[ 501-9999]:10    ← 长尾"非酋"报告 (含 1200/1400/2000/2300/2388/...)
```

**关键观察**:
- **60-90 区有 10 条命中**, 与 80 次硬保底吻合.
- **n=80 有 6 条** 真个人报告 (清洗后): "我XX只才出" 实测.
- **长尾 ≥500 有 10 条** 远超 80 保底 — 三种解释假设:
  - (a) 球数 / 总抓捕数误算成污染数 (区分困难)
  - (b) 跨家族池切换导致单池污染计数被重置, 玩家累计绝对数大
  - (c) 1.8% 早期触发的极端非酋 (但应被硬保底兜住)
- **n=1-9 (15 条)** 大概率非纯保底: 含两类:
  - 真欧皇 1-3 次出货
  - "我第 1 次抓的翼王是炫彩!" 等炫耀首抽

## 5. 三个核心 finding

### F1: 80 次保底确实是社区共识
n=80 单点 26 条原始命中 (清洗后 6 条真实测), 加上 79/81 周边的实测, 说明保底机制广泛被玩家验证.

### F2: 显著的非酋长尾
≥500 的 10 条命中里, 即便剔除明显球数语境, 仍有数条**远超 80 保底**的报告. 这意味着:
- 跨家族切换吃保底**很常见**, 玩家实际"绝对污染数"分布远比"单池保底数"宽
- 后续建模需明确区分 *单池污染计数*(理论 ≤80) vs *总污染数*(无上界)

### F3: 自报数据的固有偏差
`mechanic_qna + ball_count + other = 108 / 160` (67% 是噪音). 即便最干净的源 (B 站高赞视频评论), 提及数字 ≠ 提及保底数. 后续:
- LLM 二次抽取 (Haiku) 可能比正则更准
- 主播实测视频 (录制全过程) 是唯一无偏样本源

## 6. 数据有效性 vs 研究需求

V2 设计需要 ~500 个 (post, pity_count, family_pool) 三元组做贝叶斯模型. 目前 52 个干净点位远不够, 但流程已跑通.

**Scale-up 路径**:
- 当前 30 视频 → 扩到 100 视频, 评论数线性 → 估计 175 personal hits
- 加入贴吧 / 知乎 / TapTap (需各自爬虫) → +200
- LLM 抽取替代正则, recall 提升 ~3x → 总预期 1200+ 干净 hits

## 7. 已知 bug / 局限

1. **正则 4-digit 截断已修**, 但部分长尾 hits 的"球数 vs 污染数"语义无法区分
2. **n=1-9 区** 含 *次序*("第 1 只异色"=累积第 1) vs *单池保底*("第 1 次污染就出") 两种语义, 难自动区分
3. **L2 回复抓 5 页**, 长 L2 chain (100+ 回复) 会被截断
4. **关键词 "洛克王国 异色 概率"** 召回有限 (主要打到攻略视频), 应增加 "非酋 异色" / "出货 异色" 等口语词

## 9. LLM 二次抽取（codex / gpt-5.4）

**输入**: `data/processed/llm_candidates.jsonl` — 2,897 含数字+pity 关键词的预筛评论
**输出**: `data/processed/llm_pity_extractions.jsonl` — codex 严格判别后保留 45 条
**Rejection rate**: 98.4% — 比规则法激进得多, 但保留的更精准
**脚本**: `src/analysis/extract_llm_pity_reports.py` (codex 自动生成)

### 9.1 LLM 自动 scope 分类

| scope | 含义 | n 列表 |
|---|---|---|
| `single_pool` (30) | 单家族池真实污染计数 | 1,1,1,2,2,3,4,4,4,5,9,15,17,21,25,30,**50,50,55,56,60,60,66,67,68,69,71,81**,100,140 |
| `cross_pool_total` (8) | 跨家族切换累计 | 30, 83, 89, **90,90**, 100, **1300, 1500** |
| `unknown` (7) | 上下文不足判断 | 2, 12, 30, 60, 60, 68, 70 |

### 9.2 关键发现：F2 假设确认

**LLM 把 1300/1500 这类"非酋"自动归到 `cross_pool_total`**——证实长尾不是单池保底失效, 而是玩家**跨家族切换**导致单池计数被频繁重置, 最终累计绝对污染数远高于 80.

这意味着:
- **理论建模时**: 单池数据应在 60-90 区间形成清晰峰 (M1 硬保底假设 → median ≈ 42)
- **观测的 single_pool median = 30** (被 1-25 的"早期出货"压低), 50-71 真保底带 12 条
- 排除 n<10 后, single_pool median ≈ 50, mean ≈ 56 — 接近 M1 理论 (E[X | M1] ≈ 42)
