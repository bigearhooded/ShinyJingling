# 06 · 贝叶斯模型识别结论 — Stage 2 报告

> 数据：清洗后 single_pool 子集 N=266 (`data/processed/pity_clean.jsonl`)
> 模型：M1/M2/M3 + 4 段 posting weight $w(k)$ (V2 §2.3 修正版)
> 采样：PyMC 5.25, NUTS, 4 chains × 1500 draws

## 关键结论 1：**官方 0.02 综合概率被强烈拒绝**

⚠️ **重要修正**: 0.018 是 **玩家反推** 的 per-trial base rate, **不是官方公布**。
官方实际给出的是 **"综合概率 ≈ 0.02"**, 即每次击破污染的长期平均出货率 = $1/E[\tau]$。

| 模型 | $\hat p_0$ (后验均值) | $1/\hat E[\tau]$ (= 综合出货率) | $P(p_0 < 0.0143)^*$ | $P(p_0 < 0.018)^{**}$ |
|---|---|---|---|---|
| M1 (硬保底) | **0.092** | **0.092** | 0.00% | 0.00% |
| M2 (线性软) | **0.062** | **0.063** | 0.00% | 0.00% |
| M3 (凸软)   | **0.062** | **0.062** | 0.00% | 0.00% |

\* 0.0143 是 M1 下"综合概率 0.02"对应的 per-trial $p_0$ ($1/E[\tau] = 0.02 \Rightarrow p_0 \approx 0.0143$).
\** 0.018 是玩家圈反推的 per-trial 估计.

**所有三个模型的综合出货率后验都集中在 0.06–0.09 区间, 是官方"0.02 综合概率"的 3–4.5 倍**.

> 这是在已经做了**双向 posting bias 修正** (n=1 晒福报 + n≥78 保底投诉) 之后的结果.
> 即把"自报数据偏向极端事件"完全归还后, 真实每次击破的异色出货率仍**显著高于官方公布的 0.02**.
>
> 三种解释 (按可信度排序):
> 1. **真实概率高于公示**: 官方"0.02 综合"是营销下界, 实际工程概率约 0.06–0.09 (即 ~6–9%). 这与官方一贯保守披露习惯一致.
> 2. **Posting bias 模型比 §2.3 更复杂**: 除 n=1 / n≥78 之外, 中段 30–77 也有显著抑制 ("我刷了几十次没出"才肯 post), 会把后验 $p_0$ 推高.
> 3. **数据本身偏向"已成功"玩家**: B 站评论区聚集的是已经出过异色的玩家, 选择性 sampling, 长尾失败者沉默.
>
> 解释 1 与本研究的"非酋上限"结论 (§7) 一致 — 即使官方实际故意低报概率, 也对玩家是利好的反向欺骗.

## 关键结论 2：**数据偏好硬保底, 不是软保底**

LOO 排序：

```
       elpd_loo       elpd_diff   weight
M1     -789.25         0.00       1.00
M2     -802.48       -13.22       ~0
M3     -819.71       -30.45       ~0
```

- **M1 强 dominant** (LOO weight = 1.0)
- M2 的 $\hat k^* = 78.2$ → 线性软保底窗口仅 2 步, 几乎退化为硬保底
- M3 的 $\hat \eta = 19.3$ → 凸曲线极陡, 79 步前几乎平坦, 80 步前飙升, 也几乎是硬保底

> 这反过来**支持**了"硬保底机制"假说: 数据找不到任何"温和上升"的轨迹证据。
> 官方机制描述 "80 次必出" 是 80 次精确触发, 不是软保底渐进。

## 关键结论 3：Posting bias 量级估计

`w(k) = exp(b0 + b_first·1[k=1] + b_low·1[2≤k≤29] + b_pity·1[k≥78])`

| 参数 | 经济含义 | M2 后验均值 |
|---|---|---|
| $\hat b_\text{first}$ | log( w(n=1) / w(mid) ) — 一发入魂晒福报 | **2.3** (≈ 10×) |
| $\hat b_\text{low}$ | log( w(2..29) / w(mid) ) — 早期常规晒 | **0.09** (≈ 1.1×) |
| $\hat b_\text{pity}$ | log( w(78..80) / w(mid) ) — 保底投诉 | **2.3** (≈ 10×) |

> 中段 (30–77) 几乎被 selection 完全抹平 (`b_low ≈ 0`), 而 n=1 和 n≥78 各被超采样 ~10 倍。
> 这与 V2 §1.3 假说"凡尔赛 / 终于肝完"完全吻合。

## 模型 identifiability 警示

M1 和 M2/M3 给出的 $p_0$ 不同, 反映了一个根本不可识别性：
- 高 $p_0$ + 弱 posting bias (M1: $p_0$=0.09, $b_\text{first}$=0.66) 与
- 低 $p_0$ + 强 posting bias (M2/M3: $p_0$=0.06, $b_\text{first}$=2.3)
都能拟合数据。

LOO 选 M1 因为复杂度更低 (k_star/eta 多余), 但绝对的 $p_0$ 数值仍依赖 posting bias 假设。
**鲁棒结论**: $p_0 \in [0.05, 0.11]$ 区间, 至少 3 倍于官方 1.8%。

## 后续 Stage 4 (§1.2/3.2) 建模采用值

考虑模型 identifiability, 在策略仿真时使用：

- **保守版**: $p_0 = 0.018$ (玩家反推的 per-trial 估计, 给最差打算)
- **诚实版**: $p_0 = 0.062$ (M2/M3 后验均值)
- **乐观版**: $p_0 = 0.092$ (M1 后验均值)

并对比三种策略 (S-P/S-F/S-C) 在三种 $p_0$ 下的期望利润. 见 `data/processed/strategy_table.md`.
